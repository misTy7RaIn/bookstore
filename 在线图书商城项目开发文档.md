# 在线图书商城项目开发文档

# 一、项目概述

本项目为在线图书商城系统，采用前后端分离架构，后端基于SpringBoot + Controller + Service + Mapper + Pojo分层实现，前端基于Vue.js构建，实现了用户端购书全流程与管理员端后台管理功能，包含用户认证、图书管理、购物车、订单交易、数据可视化等核心模块。本项目由一人独立开发完成，覆盖前后端全栈开发所有内容。

# 二、开发人员与职责说明

本项目由单人独立开发完成，全面负责项目所有模块的设计、编码、测试与交付，具体职责如下：

- 后端核心层：SpringBoot项目搭建、用户认证、邮箱验证码注册、权限拦截、数据库连接池与工具类封装

- 后端图书层：图书查询、详情展示、搜索联想、CRUD与OSS文件上传（基于SpringBoot实现接口）

- 后端交易层：购物车、订单管理、收藏功能、收货地址管理（基于SpringBoot接口开发）

- 前端用户端：前台Vue页面、Axios请求封装、表单校验与交互

- 前端管理端：后台Vue页面、ECharts数据可视化、权限判断

# 三、技术栈

## 3.1 后端技术

- 核心框架：SpringBoot 3.2.5（替代Java Servlet，简化配置，实现接口开发）

- 分层架构：Controller + Service + Mapper + Pojo（基于SpringBoot规范搭建）
  - Controller：接收请求，返回响应
  - Service：处理业务逻辑
  - Mapper：数据库持久化操作
  - Pojo：JavaBean实体类，对应数据库表结构

- 数据库：MySQL 8.0

- ORM框架：MyBatis（注解 + XML映射）

- 安全：Spring Security Crypto（BCrypt密码加密）

- 邮件：Spring Boot Mail + QQ邮箱 SMTP（邮箱验证码）

- 对象存储：阿里云 OSS（图书封面上传）

- 工具类：Token生成与校验、SpringBoot拦截器（替代LoginFilter实现权限拦截）

- 其他：文件上传（MultipartFile）、统一编码处理、全局异常处理、防暴力破解延迟响应

## 3.2 前端技术

- 框架：Vue 3

- 状态管理：Pinia

- 网络请求：Axios

- 数据可视化：ECharts

- 构建工具：Vite

- 样式：CSS / Less

# 五、核心技术实现细节

## 5.1 权限控制

- 通过SpringBoot拦截器（LoginInterceptor）拦截未登录请求，从请求头读取Token校验身份，配置拦截路径与放行路径

- 基于TokenUtil生成用户令牌，登录时持久化到数据库user.token字段，拦截器中查询校验

- 区分三种角色：1-普通用户，2-管理员，3-超级管理员。管理员端路由需额外校验角色权限，超级管理员拥有创建/删除管理员账号权限

## 5.2 数据库连接

- 在SpringBoot项目中集成HikariCP连接池（SpringBoot默认），通过application.yml配置连接参数，提升性能与稳定性

- 使用MyBatis作为ORM框架，Mapper接口 + XML映射文件操作数据库

## 5.3 搜索联想

- 后端基于MySQL模糊查询实现关键词匹配，通过SpringBoot Controller暴露接口，返回联想结果

- 前端通过防抖处理减少请求频次，调用SpringBoot后端接口，实时展示联想结果

## 5.4 数据可视化

- 管理员端集成ECharts，展示图书数量、订单数量、用户数量、总销售额、分类图书分布、日订单趋势等统计图表

- 后端基于SpringBoot DashboardController开发聚合查询接口（DashboardMapper），封装图表所需数据格式，供前端ECharts调用渲染

## 5.5 邮箱验证码

- 集成Spring Boot Mail + QQ邮箱SMTP服务，注册时向用户邮箱发送6位随机验证码

- 验证码存储于内存（VerificationStore），有效期5分钟，同邮箱60秒冷却机制防刷

## 5.6 安全防护

- 密码使用BCrypt（Spring Security Crypto）加密存储，user表同时保留raw_password字段

- 登录/注册失败时延迟2秒响应（Thread.sleep），防暴力破解

- Token持久化存储，服务端可控失效

## 5.7 阿里云OSS文件上传

- 图书封面通过阿里云OSS对象存储上传，OssService封装上传逻辑

- OSS配置（endpoint、accessKey、bucket）在application.yml中管理，返回公开访问URL

# 六、数据库表结构设计

## 6.1 数据库总览

本项目使用MySQL数据库，命名为bookstore，包含以下核心表：

- user：用户信息表
- category：图书分类表
- book：图书信息表
- cart：购物车表
- orders：订单主表
- order_item：订单明细表
- favorite：收藏表
- address：收货地址表

## 6.2 表结构详情

### 6.2.1 用户表（user）

存储用户/管理员的账号、密码、角色、邮箱等信息。

| 字段名         | 类型           | 约束                                                           | 说明                         |
| -------------- | -------------- | -------------------------------------------------------------- | ---------------------------- |
| user_id        | INT            | PRIMARY KEY AUTO_INCREMENT                                     | 用户ID（主键）               |
| username       | VARCHAR(50)    | NOT NULL UNIQUE                                                | 用户名（唯一）               |
| password       | VARCHAR(100)   | NOT NULL                                                       | 密码（BCrypt加密存储）       |
| phone          | VARCHAR(20)    | NOT NULL UNIQUE                                                | 手机号（唯一）               |
| email          | VARCHAR(100)   | UNIQUE                                                         | 邮箱地址（唯一）             |
| email_verified | TINYINT        | NOT NULL DEFAULT 0                                             | 邮箱验证状态：0-未验证，1-已验证 |
| role           | TINYINT        | NOT NULL DEFAULT 1                                             | 角色：1-普通用户，2-管理员，3-超级管理员 |
| token          | VARCHAR(255)   |                                                                | 登录令牌（持久化存储）       |
| raw_password   | VARCHAR(100)   |                                                                | 原始密码（明文备份）         |
| create_time    | DATETIME       | NOT NULL DEFAULT CURRENT_TIMESTAMP                             | 创建时间                     |
| update_time    | DATETIME       | NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间                     |

### 6.2.2 图书分类表（category）
