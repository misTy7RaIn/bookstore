# 在线图书商城 —— 本地部署指南

## 一、环境要求

| 软件 | 版本要求 | 说明 |
| --- | --- | --- |
| JDK | 17+ | 后端运行环境 |
| Maven | 3.6+ | 后端构建工具 |
| Node.js | 18+ | 前端运行环境 |
| npm | 9+ | 前端包管理 |
| MySQL | 8.0+ | 数据库 |

## 二、MySQL 数据库配置

### 2.1 创建用户与数据库

打开 MySQL 命令行工具，依次执行：

```sql
-- 1. 创建用户（与项目配置保持一致）
CREATE USER 'bookstore'@'localhost' IDENTIFIED BY '123456';

-- 2. 创建数据库
CREATE DATABASE bookstore
    DEFAULT CHARACTER SET utf8mb4
    DEFAULT COLLATE utf8mb4_unicode_ci;

-- 3. 授权
GRANT ALL PRIVILEGES ON bookstore.* TO 'bookstore'@'localhost';
FLUSH PRIVILEGES;
```

### 2.2 导入数据

项目根目录下有完整的数据库 dump 文件，一键导入即可（含建表 + 数据）：

```bash
mysqldump_path -u bookstore -p123456 bookstore < G:\bookstore\bookstore_dump.sql
```

> 如果 `mysqldump` 不在系统 PATH 中，请替换 `mysqldump_path` 为你的 MySQL 安装 bin 目录下的 `mysql.exe` 路径，例如：
> ```
> "G:\MySQL\bin\mysql" -u bookstore -p123456 bookstore < G:\bookstore\bookstore_dump.sql
> ```

## 三、后端部署

### 3.1 配置文件

打开 `bookstore-backend/src/main/resources/application.yml`，确认数据库连接信息与本地一致：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/bookstore?useUnicode=true&characterEncoding=UTF-8&serverTimezone=Asia/Shanghai
    username: bookstore
    password: 123456
    driver-class-name: com.mysql.cj.jdbc.Driver
```

> 如果 MySQL 默认端口是 3306 且用户名密码为 `bookstore/123456`，则无需修改。

### 3.2 启动后端

```bash
cd G:\bookstore\bookstore-backend

# 使用 Maven 构建并启动
mvnw spring-boot:run

# 如果 mvnw 不可用，使用系统 Maven
mvn spring-boot:run
```

启动成功后，后端运行在 **http://localhost:8080**。

## 四、前端部署

### 4.1 用户端（bookstore-user）

```bash
cd G:\bookstore\bookstore-user

# 首次运行需安装依赖
npm install

# 启动开发服务器
npm run dev
```

启动后访问 **http://localhost:5173**。

### 4.2 管理端（bookstore-admin）

```bash
cd G:\bookstore\bookstore-admin

# 首次运行需安装依赖
npm install

# 启动开发服务器
npm run dev
```

启动后访问 **http://localhost:3000**。

### 4.3 修改 API 代理地址（重要）

两个前端项目的 Vite 配置中，API 代理默认指向远程服务器。本地部署时需要改为本地后端地址。

**bookstore-user/vite.config.js**（第 18 行）：
```js
proxy: {
  '/api': {
    target: 'http://localhost:8080/',  // 改为本地地址
    changeOrigin: true
  }
}
```

**bookstore-admin/vite.config.js**（第 18 行）：
```js
proxy: {
  '/api': {
    target: 'http://localhost:8080/',  // 改为本地地址
    changeOrigin: true
  }
}
```

## 六、常见问题

**Q: 启动后端报 "Access denied for user"**
A: 检查 MySQL 用户 `bookstore` 是否存在，密码是否为 `123456`。参考第二章重新创建用户并授权。

**Q: 启动后端报 "Unknown database 'bookstore'"**
A: 数据库未创建，执行第二章的建库语句或直接导入 `bookstore_dump.sql`。

**Q: 前端页面打开但数据加载失败**
A: 检查 `vite.config.js` 中的 proxy target 是否已改为 `http://localhost:8080/`。

**Q: 端口被占用**
A: 后端默认 8080，用户端 5173，管理端 3000。如需修改：
- 后端：修改 `application.yml` 中的 `server.port`
- 前端：修改对应 `vite.config.js` 中的 `server.port`
